Reasoning GP · Quality incident report

Why Ask got the postpartum contraceptive-patch question wrong — and the plan to stop it happening

Prepared 7 July 2026 · Question: “Can I prescribe a contraceptive patch for a 30-year-old who gave birth 8 weeks ago?” · Two runs, two different users, two different answers — one clinically wrong.

1 · The clinical verdict first

The correct answer (UKMEC 2025, published December 2025 by the CoSRH — formerly FSRH):

2 · Root causes — five, all identified in our own pipeline

Cause 1  The web search is blocked from the one source that holds the answer

Ask’s live web search is restricted to an allow-list of trusted UK domains (NICE, SIGN, BMJ Best Practice, gov.uk, RCOG, BASHH…). fsrh.org / cosrh.org is not on the list. UKMEC is published by CoSRH (formerly FSRH) — so for any contraception-eligibility question, the tool literally cannot reach the source of truth. User B’s answer even confessed it: “the search results don’t give me the specific UKMEC figures I need.” When retrieval fails, the model falls back on training memory — which is where the WHO/UK mix-up and the stale “2016” citation came from.

Cause 2  No library note covers postpartum contraception, so grounding had nothing to offer

Ask’s first line of defence is the site’s own curated library (~975 notes). We have a contraception method-chooser tool and SCA cases that mention UKMEC — but no article on postpartum/breastfeeding contraceptive eligibility for the retrieval engine to hand the model. High-traffic, high-stakes topics must exist as curated notes; the model should be assembling our vetted content, not recalling its own.

Cause 3  Inconsistency between users is built into the current setup

  • No temperature is set on the AI call, so it runs at the creative default — two identical questions can legitimately produce different answers.
  • The answer cache only matches exact wording. User A typed “gave birth 8 weeks ago”, User B “who give birth 8 weeks ago” — different cache keys, so each got an independent fresh generation instead of one vetted answer.
  • The free backup model. When Anthropic credit runs out, Ask silently falls back to a free open-source model (with a small ⚠️ note). That model is markedly weaker clinically. If one of the two runs was a fallback answer, that alone explains the quality gap.

Cause 4  Model “thinking out loud” leaked into the answer

“The search results don’t give me the figures I need, I’ll answer from my firm knowledge” is internal narration. It should never be shown — it erodes trust, and worse, it flags exactly the situation where the answer should have been withheld or hedged, not delivered with a confident table.

Cause 5  Tone: orders instead of recommendations

The answer framing tells the model to “lead with the single most useful next step”, which comes out as imperatives (“Stop the drug. Refer.”). For a decision-support tool used by qualified GPs, the register should be advisory: “Recommended / consider / options are…”, explicitly supporting — not replacing — the clinician’s judgement. This needs one added framing rule; it applies to every answer, not just this one.

4 · Do we need a bigger AI model?

Short answer: not yet — and it would not have fixed this incident.

  • Both failures were evidence-supply problems, not reasoning problems. The strongest model on earth cannot cite UKMEC 2025 if the search allow-list blocks it and the library doesn’t contain it — it will just hallucinate more fluently. Fixes 1–4 attack the actual cause.
  • The current model (Sonnet-class) is the right tier for grounded Q&A. The step up (Opus-class) costs roughly 5× per token: at our trimmed context sizes that means moving from roughly 4–6p per fresh question to ~20–30p. Cached/vetted answers stay at ~0p either way — which is another reason the vetted cache is the best money you can spend.
  • Sensible path: implement fixes 1–7, re-run the full test bank (add the contraception set), and only if graded accuracy still disappoints, A/B the bigger model on the test bank before paying for it in production.

5 · On “zero percent chance of mistakes”

Honestly: no AI system — and no clinician — reaches literal zero. What we can engineer is a system where the high-traffic questions are answered from vetted, owner-approved answers (deterministic, identical for every user), the long tail is grounded in the live guideline sources with clickable citations, and anything the system cannot verify is flagged as unverified instead of asserted. That is what fixes 1–7 build. The disclaimer line (“educational support — verify against live NICE CKS/BNF”) remains essential for governance either way.
